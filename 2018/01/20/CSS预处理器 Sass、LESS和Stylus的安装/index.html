CSS预处理器 Sass、LESS和Stylus的安装

2018年01月20日

什么是CSS预处理器

CSS预处理器定义了一种新的语言，其基本思想是用一种专门的编程语言，为CSS增加了一些编程的特性，将CSS作为目标生成文件，然后开发者就只要使用这种语言进行编码工作。通俗的说，CSS预处理器用一种专门的编程语言，进行Web页面样式设计，然后在编译成正常的CSS文件，以供项目使用。CSS预处理器为CSS增加了一些基本特性，无需考虑浏览器的兼容性问题，可以在CSS中使用变量、简单的逻辑程序、函数等等，可以让CSS更加简洁、适应性更强、可读性更佳，更易于代码的维护等诸多好处。

CSS预处理技术已经非常的成熟，而且也涌现出了很多不同的CSS预处理器语言，比如：Sass(SCSS)、LESS、Stylus、Turbine、Swithch CSS、DT CSS等。如此之多的CSS预处理器，那么选择哪种CSS预处理器那？

到目前为止，在众多优秀的CSS预处理器语言中就属Sass、LESS、Stylus最优秀，讨论的也多，对比的也多，本文将会从他们的产生的背景、安装、使用语法、异同等几个对比像你介绍这三款CSS预处理器语言。

##Sass、LESS和Stylus的背景介绍

Sass背景介绍

Sass是对CSS (层叠样式表) 的语法的一种扩充，诞生于2007年，最早也是最成熟的一款CSS预处理器语言，它可以使用变量、嵌套、混入、函数等功能，可以有效弹性的写出CSS。Sass最后会编译出合法的CSS让浏览器使用，也就是它更像一种极简单的动态语言。

其实现在的Sass已经有了两套语法规则：一个依旧是用缩进作为分隔符来区分代码块；另一套规则和CSS一样采用了大括号( {} )作为分隔符。后一种语法规则又名SCSS，在Sass3之后的版本都支持这种语法规则

LESS的官网

LESS背景介绍

2009年来源的一个项目，受Sass的影响较大，但又使用CSS的语法，让大部分开发者和设计师更容易上手。LESS提供了很多种方式能平滑的将写好的代码转化成标准的CSS代码，在很多主流的框架和工具中已经能经常看到LESS的身影了(例如Twitter的Bottstrap框架就使用了LESS)

根据维基百科上的介绍，其实LESS是Alexis Sellier受Sass的影响创建的一个开源项目。当时SASS采用了缩进作为分割符来区分代码块，而不是CSS中广为使用的大括号( {} )。为了让CSS现有的用户使用起来更加方便，Alexis开发的LESS并提供了类似CSS的书写功能。

LESS的官网

Stylus背景介绍

Stylus，2010年产生，来自于Node.js社区，主要用来给Node项目进行CSS预处理支持，在此社区内有一定支持者，在广泛的意义上人气还完全不如Sass和LESS。

Stylus被称为是一种革命性的新语言，提供一个高效、动态、和使用表达方式来生成CSS，以供浏览器使用。Stylus同时支持缩进和CSS常规样式书写规则。
Stylus官网

##Sass、LESS和Stylus的安装
因为我本人的电脑是mac，所以我只讲mac的安装。

Sass的安装

Sass是用Ruby语言写的，但是两者的语法没有关系，所以学 Sass 不用学 Ruby，只是必须先安装Ruby，然后再安装Sass
Linux和Mac已自带Ruby，不用再安装。

验证Ruby是否安装成功

ruby -v

或者

ruby --version

如果输出Ruby版本号，则便是安装成功

安装SASS

在命令行输入下面的命令

gem install sass

如果是Mac的话可能会遇到这样的问题：问题1：

ERROR: While executing gem ... (Gem::FilePermissionError)
You don't have write permissions for the /Library/Ruby/Gems/2.0.0 directory.   
                这是因为没有管理员权限.png

执行 sudo gem install sass -> 回车 -> 输入密码 即可：
密码是管理员密码(我的是Mac的开机密码) 

sudo gem install sass

紧接着可能会遇到第2个问题 （我执行的时候没有这个错误，直接成功了）

ERROR: While executing gem ... (Errno::EPERM)
Operation not permitted - /usr/bin/sass

这样Sass就安装成功了。检测是否安装成功

sass -v

或者 

sass --version

如果输出Sass版本号，则表示安装成功

LESS的安装

在命令行输入下面的命令

npm i less -g

Stylus的安装

Stylus的安装和LESS在服务器的安装很相似，同样依赖于Node JS的环境，也就是说需要先安装Node JS.

在命令行输入下面的命令

npm install -g stylus

下载成功，从版本信息就可以看出，Stylus作为一种新型语言相比于less，Sess，并不如何出众

获得相关的命令行支持

stylus -h


这样就算安装完Stylus了，也可以正常使用Stylus
